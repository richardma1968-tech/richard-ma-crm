Deploy Checklist

Use this every time you move a new HTML upgrade to GitHub and Vercel.
1. Test locally first
Open the new HTML file locally and test:
• Add Lead
• Run Automation
• Daily Mode
• Kanban Board
• Backup JSON
2. Confirm version details
Check that the header shows the correct:
• Version
• Updated date
• Updated time
3. Prepare the file
Rename the final file to:
index.html
Make sure it is a real .html file, not .txt
4. Update GitHub
Open your repo → open index.html → click Edit → replace all code → Commit changes
5. Let Vercel deploy
Open Vercel and wait for the new deployment to finish.
6. Verify live
Open the live URL and hard refresh:
Ctrl + Shift + R
Confirm:
• Version matches
• Updated time matches
• Deployment status says Live
• No version mismatch warning
7. Final live test
Test:
• Add Lead
• Run Automation
• Daily Mode
• Follow-Up Due Today
• Kanban stage movement
8. Safety backup
Click Backup JSON after deployment and save it in your CRM Backups folder.
9. Working file rule
Keep one master local file:
CRM Master > index.html
Archive older versions in:
CRM Backups
